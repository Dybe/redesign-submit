Responsive Redesign

For this project, I practiced workflow of redesigning a simple website. I did so by performing usability analysis on a website of interest in order to identify flaws. I then created low and high fidelity prototypes for desktop, mobile and tablet screen sizes before building a responsive website according to the prototypes.

The website:

I chose to redesign the homepage of the official website of Kentish Town City Farm, https://ktcityfarm.org.uk/.

The farm is a community charity based in London’s borough of Camden.It endeavours to connect Londoners with nature and animals through an educational program to schools, therapeutic riding to those with special needs, work experience for youth and farm activities. I chose the farm’s website because it allowed me to reconnect with a neighborhood I care for deeply.

Usability Issues

The webpage has a myriad of usability issues highlighted below:

  • Inconsistent sizing and organization of ‘boxes’ at the bottom of the page. Map inside the “We are here” box extends outside the box.

        

  • Typography: Font used is legible but somewhat decorative. Whether this is a legitimate usability issue for this specific website could be debatable, but I will treat it as such.

  • Inconsistent layout when the website is accessed via mobile or tablet.
  1. On Mobile:
  1. Image icons linking the website to the farm’s Facebook and Twitter overlay and hide some of the text in the header.
  2. Navigation bar is hidden. Although it is expandable, it does not indicate that the user is looking at the homepage
  1. On Tablets:
  1. Website’s header text is reduced to “City Farm”
  2. The “a” in City Farm is covered by image icon links to Facebook and Twitter
  3. Header Image is cropped
  4. Information boxes at the bottom are in even greater disarray

                                        

Accessibility

WebAIM WAVE Concerns: Very low contrast on privacy policy and donation links; Journey planner selections are not labeled; Text on the site skipped a heading level as it lacks a h1 heading; and has redundant attribute text for the navigation bar titles.

                        

A screen reader was able to interpret most of the items on the page except for the buttons and icons links to the farm’s social media pages. The screen reader did not notice them.

Visual Redesign

I endeavoured to solve the above issues by coming up with low-fidelity  and high-fidelity prototypes (using balsamiq and figma respectively ) together with a Visual Design Style Guide before building the webpage using HTML and CSS. The prototypes are annotated to explain how the changes solve the issues presented and how the changes can be built.

Conclusions and Learnings

This project gave me an opportunity to experience the end-to-end web (re)design process. I was able to practice and solidify my understanding of usability testing, prototyping and building responsive websites using HTML and CSS
